clion配置gdal遇到的问题及解决办法

ytkz2024-07-252024-09-09

因为经常使用pycharm，对Jetbrains家的产品有一些莫名的好感。

所以在写C++想使用一下他们家的clion ide 软件。

首先遇到的第一个问题就是，怎么在clion使用gdal。

这篇文章是记录clion配置gdal遇到的问题及解决办法。

步骤1: 安装Vcpkg

  1. 下载Vcpkg: 首先，从GitHub上克隆Vcpkg仓库到你的本地机器上。打开终端或命令提示符，执行以下命令:

    Bash

    1git clone https://github.com/microsoft/vcpkg.git

  1. 初始化Vcpkg: 进入vcpkg目录并初始化vcpkg环境。

    Bash

    1cd vcpkg
2./bootstrap-vcpkg.sh  # 对于Linux/macOS
3./bootstrap-vcpkg.bat  # 对于Windows

步骤2: 安装GDAL

  1. 安装GDAL: 使用Vcpkg安装GDAL库。你可以指定特定的版本或者使用默认的最新版本。

    Bash

    vcpkg install gdal  # 安装默认版本

如果你需要指定版本，可以查阅Vcpkg的列表以确定可用版本号，然后像这样安装：

Bash

vcpkg install gdal:x64-windows  # Windows平台，指定架构

可能遇到的错误：

vcpkg安装libpq失败。libpq是postgresql是别名。

解决办法。

在visual studio 的语音包选项中，去除中文，只保留英语。

然后重新安装libpq。

步骤3: 配置CLion

  1. 创建或打开项目: 在CLion中创建一个新的C++项目或打开现有的项目。

  2. CMakeLists.txt配置: 在项目的CMakeLists.txt文件中，添加以下内容来指定使用Vcpkg安装的GDAL库。确保路径正确指向你的Vcpkg安装目录。

    Cmake

    1set(CMAKE_TOOLCHAIN_FILE "{path_to_vcpkg}/scripts/buildsystems/vcpkg.cmake" CACHE STRING "" FORCE)
2find_package(GDAL REQUIRED)
3include_directories(${GDAL_INCLUDE_DIRS})
4target_link_libraries(your_target_name ${GDAL_LIBRARIES})

替换{path_to_vcpkg}为你的Vcpkg实际安装路径，your_target_name为你的CMake目标名称。

  1. 刷新CMake: 在CLion中，点击”File” -> “Reload CMake Project”或者使用快捷键，让CLion重新加载CMake配置。

  2. 编译与运行: 现在，你应该能够在代码中包含GDAL头文件并链接到GDAL库进行编译和运行了。

注意事项

  • 确保你的CLion版本与你的操作系统兼容，并且支持CMake项目。
  • 如果你的CLion安装路径含有空格，可能导致Vcpkg使用时出现问题，根据之前的提示，可能需要单独安装CMake并配置CLion使用该独立安装的CMake。
  • 如果遇到编译问题或找不到库，请检查GDAL是否正确安装，以及CMakeLists.txt中的路径是否正确。
  • 保持Vcpkg更新，以获取GDAL的最新版本和支持。
