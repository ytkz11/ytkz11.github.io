clion配置gdal遇到的问题及解决办法

ytkz2024-07-252024-09-09

因为经常使用pycharm，对Jetbrains家的产品有一些莫名的好感。

所以在写C++想使用一下他们家的clion ide 软件。

首先遇到的第一个问题就是，怎么在clion使用gdal。

这篇文章是记录clion配置gdal遇到的问题及解决办法。

安装WSL和配置号ubantu

教程：https://blog.csdn.net/weixin_57367513/article/details/135001273

安装 GDAL

sudo add-apt-repository ppa:ubuntugis/ppa && sudo apt-get update
sudo apt-get update
sudo apt-get install gdal-bin
sudo apt-get install libgdal-dev

如果一切正确，已成功安装 GDAL！

在 CLion 上开始使用 GDAL 之前，需要完成以下两个步骤。

安装 CMake

要安装 CMake，请在 Ubuntu 终端上运行以下命令。

安装CMake所依赖的库：

sudo apt-get install build-essential libssl-dev

进入 tmp 目录：

cd /tmp

下载源代码（将路径改为最新版本的路径）：

wget https://github.com/Kitware/CMake/releases/download/v3.28.3/cmake-3.28.3.tar.gz

完成后，提取文件：

tar -zxvf cmake-3.28.3.tar.gz

进入解压的文件夹：

cd cmake-3.28.3

编译并安装CMake：

./bootstrap

这可能需要一些时间，完成后您将看到以下内容。

现在运行以下命令来实现：

make

这一步也需要一些时间。完成后，你终于可以安装它了：

sudo make install

最后，为了检查安装是否成功，您可以查看 CMake 版本：

cmake --version

安装 GDB¶

GDB 是 C++ 的调试器，您将需要它在 CLion 中连接到 WSL。

如果您在 CMake 之后立即安装 GDB，则可能仍位于cmake-3.20.0Ubuntu 终端上的文件夹中。首先，返回主目录：

cd ../..

使用以下两个命令安装GDB：

sudo apt-get update
sudo apt-get install gdb

如果此方法对您不起作用，您也可以按照本[页面]上第二个选项的说明通过源代码进行安装。

您可以通过检查 GDB 版本来验证安装：

gdb --version

一旦安装了 GDB，就拥有了在 CLion 中连接 WSL 所需的一切！

偏门技巧

wsl 控制台与 win10 之间复制粘贴的实现

ubuntu 控制台内部
复制：按住鼠标左键选择，松开选定
粘贴：点击鼠标右键

win 复制到 ubuntu 粘贴
win 复制：Ctrl + C
ubuntu粘贴：点击鼠标右键

ubuntu 复制到 win 粘贴
ubuntu 复制：Shift + Ctrl + C
win 粘贴：Ctrl + V

连接到CLion

从file转到setting或直接按 Ctrl+Alt+S。

在Build、Execution、Deployment下，转到Toolchains。然后，单击 + 号添加新工具链并选择 WSL。

它应该会自动检测所有文件（Toolset、CMake、Debugger 等）。但是，如果您看到无法找到 CMake 的错误，请单击 CMake 右侧的“浏览”并选择正确的文件。您可以在 下找到它/usr/local/bin/cmake。

完成此步骤后，你可能会在 C++ 编译器下看到以下警告：测试 cmake 运行已完成并出现错误。
在这种情况下，请在 Ubuntu 终端上输入以下命令重新启动 WSL：

wsl.exe -t Ubuntu

这将关闭终端。只需在您的设备上搜索 Ubuntu 即可再次打开它。现在，如果您再次检查 CLion，您应该不会再看到警告。

最后，不要忘记将 WSL 置于列出的工具链顶部，使其成为默认工具链。您可以通过将 WSL 拖到顶部或使用箭头来执行此操作。

现在可以在 Windows 上使用 CLion 的 GDAL了！
